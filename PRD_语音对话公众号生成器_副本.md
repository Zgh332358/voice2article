## 一、产品概述

### 1.1 产品简介
一款支持**语音对话创作**与**文档库素材复用**的公众号文章智能生成工具。用户可以通过语音与AI Agent自然对话来构思文章，也可以从个人文档库中选取素材进行参考生成，最终输出风格一致的公众号排版内容。

### 1.2 产品定位
- **目标用户**: 微信公众号运营者、内容创作者、自媒体人
- **核心价值**: 将灵感对话→成文效率提升3-5倍，保持内容风格一致性
- **差异化**: 市面上唯一同时支持「语音对话创作」+「个人文档库管理」+「公众号专用生成」三合一的产品

### 1.3 用户痛点
| 痛点 | 当前解决方案 | 本产品解决方式 |
|------|-------------|---------------|
| 写作灵感转文字慢 | 手动打字/录音后转文字 | 实时语音对话，边说边生成 |
| 风格不一致 | 人工参考旧文章 | 文档库学习+风格控制器 |
| 素材查找困难 | 散落各处的收藏夹 | 统一文档库+智能检索 |
| 排版耗时 | 手动调整格式 | AI自动排版 |

### 4.2 模块一：语音对话系统

#### 4.2.1 功能描述
用户通过语音与AI Agent进行自然对话，系统实时转写并理解用户意图。

#### 4.2.2 功能列表

| 功能点 | 详细说明 | 优先级 |
|--------|---------|--------|
| **F1.1 实时语音输入** | 支持麦克风实时录音，边说边转文字 | P0 |
| **F1.2 录音文件上传** | 支持上传音频文件（mp3/wav/m4a）进行转写 | P0 |
| **F1.3 实时转写展示** | 语音→文字实时展示在对话界面，支持编辑修正 | P0 |
| **F1.4 多轮对话管理** | 保持上下文记忆，支持追问、修改、补充 | P0 |
| **F1.5 意图识别** | 自动识别用户指令类型：生成文章/修改/查询文档库 | P0 |
| **F1.6 语音指令快捷操作** | 支持"生成文章"、"参考XX文档"、"重写"等快捷指令 | P1 |
| **F1.7 语音播放反馈** | 生成完成后可选语音播报摘要 | P2 |

#### 4.2.3 交互流程
```
用户点击录音按钮
    ↓
开始实时转写（显示波形图）
    ↓
用户口述内容 → 实时显示文字
    ↓
用户说"生成文章"或点击生成按钮
    ↓
系统提取对话核心内容
    ↓
进入生成模式（可选择：纯对话/纯文档/混合）
```

#### 4.2.4 技术选型建议
- **STT服务**: 阿里云ASR / 腾讯云语音识别 / OpenAI Whisper API
- **VAD（语音活动检测）**: WebRTC VAD 或自建模型
- **降噪处理**: RNNoise算法 或 服务商自带降噪

### 4.4 模块三：内容生成引擎

#### 4.4.1 功能描述
根据用户输入（对话内容 / 文档库内容 / 混合）生成公众号文章。

#### 4.4.2 三种生成模式

| 模式 | 触发条件 | 数据源 | 适用场景 |
|------|---------|--------|---------|
| **模式A：纯对话生成** | 用户说"生成文章" | 当前对话历史 | 即兴创作、灵感记录 |
| **模式B：文档库生成** | 用户选择文档 → 点击生成 | 选中的文档内容 | 参考范文、风格模仿 |
| **模式C：混合生成** | 用户说"参考XX文档生成" | 对话+选中文档 | 既有想法又要参考素材 |

#### 4.4.3 功能列表

| 功能点 | 详细说明 | 优先级 |
|--------|---------|--------|
| **F3.1 内容提取** | 从对话中提取核心观点、关键信息 | P0 |
| **F3.2 风格学习** | 分析文档库样本的语言风格、结构模板 | P0 |
| **F3.3 文章生成** | LLM生成完整文章（标题、大纲、正文） | P0 |
| **F3.4 风格控制** | 可调整语气、篇幅、结构模板 | P1 |
| **F3.5 多文档融合** | 将多篇文档内容智能融合成一篇 | P1 |
| **F3.6 人工干预编辑** | 生成后支持在线修改、段落重写 | P0 |
| **F3.7 生成历史** | 保存生成记录，支持回滚对比 | P2 |
| **F3.8 生成质量评估** | 自动评估生成内容的连贯性、原创性 | P2 |

## 六、数据模型设计

### 6.1 核心数据表

#### 6.1.1 用户表 (users)
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY,
    email VARCHAR(255) UNIQUE,
    phone VARCHAR(20),
    nickname VARCHAR(50),
    avatar_url TEXT,
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```

#### 6.1.2 文档库表 (documents)
```sql
CREATE TABLE documents (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    title VARCHAR(500),
    content TEXT,
    source_type ENUM('upload', 'url', 'manual'),
    source_url TEXT,
    file_type VARCHAR(10), -- pdf/docx/txt/html
    tags JSON,
    category VARCHAR(50),
    style_features JSON, -- 风格特征向量化结果
    embedding_vector VECTOR(1536), -- 向量数据库字段
    created_at TIMESTAMP,
    updated_at TIMESTAMP,
    is_deleted BOOLEAN DEFAULT FALSE
);
CREATE INDEX idx_documents_user ON documents(user_id);
CREATE INDEX idx_documents_tags ON documents USING GIN(tags);
```

#### 6.1.3 对话历史表 (conversations)
```sql
CREATE TABLE conversations (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    title VARCHAR(200),
    mode ENUM('dialogue', 'document', 'hybrid'),
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

CREATE TABLE messages (
    id UUID PRIMARY KEY,
    conversation_id UUID REFERENCES conversations(id),
    role ENUM('user', 'assistant', 'system'),
    content TEXT,
    audio_url TEXT, -- 语音文件URL
    referenced_doc_ids JSON, -- 引用的文档ID列表
    created_at TIMESTAMP
);
```

#### 6.1.4 生成记录表 (generations)
```sql
CREATE TABLE generations (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),
    conversation_id UUID REFERENCES conversations(id),
    source_mode ENUM('dialogue', 'document', 'hybrid'),
    source_data JSON, -- 来源数据（对话摘要/文档ID列表）
    generated_content TEXT,
    title VARCHAR(500),
    word_count INT,
    style_profile JSON, -- 使用的风格配置
    status ENUM('draft', 'published', 'deleted'),
    created_at TIMESTAMP,
    published_at TIMESTAMP
);
```

## 十一、风险与应对

### 11.1 高风险项
| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| 生成内容风格不匹配 | 高 | 中 | 准备30+篇样本用于few-shot；预留fine-tuning接口 |
| 语音识别在嘈杂环境准确率低 | 高 | 中 | 选择支持降噪的STT服务；增加语音预处理模块 |
| LLM生成内容空洞/泛泛而谈 | 中 | 中 | 设计对话引导机制，强制提取关键信息点 |
| 文档检索不准导致生成引用错误 | 高 | 中 | 两阶段检索（语义+关键词）+人工确认环节 |

### 11.2 技术债预警
- 初期采用few-shot快速验证，后期可能需微调模型
- 向量检索初期可降级为关键词检索，后续迭代
- 并发量小的情况下可单体部署，后续拆分为微服务

### 11.3 成本控制
- STT费用：按量计费，设置单日限额
- LLM费用：采用缓存机制（相似query复用结果）
- 存储费用：定期归档冷数据

### C. 术语表
- **STT**: Speech-to-Text，语音转文字
- **RAG**: Retrieval-Augmented Generation，检索增强生成
- **LLM**: Large Language Model，大语言模型
- **Few-shot**: 少样本学习，通过示例提示模型
- **Fine-tuning**: 模型微调，用特定数据训练模型
- **VAD**: Voice Activity Detection，语音活动检测
